# 📦 Tuples in Python

## 🔶 What is a Tuple?

Think of a tuple like a **sealed envelope**. Once you put things inside and seal it, you can look at what's inside but you **cannot change, add, or remove** anything. The contents are locked in forever.

```python
# Once created — it stays exactly like this
pakistan_provinces = ("Punjab", "Sindh", "KPK", "Balochistan", "Gilgit")
print(pakistan_provinces)
# Output: ('Punjab', 'Sindh', 'KPK', 'Balochistan', 'Gilgit')
```

> 💡 **Four things that define a tuple:**
> 🔻 **Ordered** — items stay in the position you put them  
> 🔻**Immutable** — cannot be changed after creation  
> 🔻**Allows duplicates** — same value can appear multiple times
> 🔻**Mixed types** — can hold integers, strings, floats, even other tuples

## 🔶 Creating a Tuple

### Method 1 — Parentheses `()` (most common):
```python
coordinates  = (24.8607, 67.0011)       # Karachi GPS coordinates
rgb_color    = (255, 165, 0)            # Orange in RGB
student_info = ("Ayesha", 21, "CS", 3.8)  # mixed types
print(coordinates)
print(rgb_color)
print(student_info)
```

### Method 2 — Without parentheses (tuple packing):
```python
# Parentheses are optional — commas create the tuple
point = 10, 20, 30
print(point)          # (10, 20, 30)
print(type(point))    # <class 'tuple'>
```

### Method 3 — `tuple()` constructor:
```python
# Convert a list into a tuple
marks_list  = [88, 92, 75, 96, 81]
marks_tuple = tuple(marks_list)
print(marks_tuple)    # (88, 92, 75, 96, 81)

# Convert a string into a tuple of characters
letters = tuple("Python")
print(letters)        # ('P', 'y', 't', 'h', 'o', 'n')
```

### ⚠️ The single element trap:
```python
# Wrong — Python treats this as just an integer in parentheses
not_a_tuple = (42)
print(type(not_a_tuple))    # <class 'int'>

# Right — add a trailing comma
is_a_tuple = (42,)
print(type(is_a_tuple))     # <class 'tuple'>

# Also right — without parentheses
also_a_tuple = 42,
print(type(also_a_tuple))   # <class 'tuple'>
```

> ⚠️ A single item tuple **must** have a trailing comma. The comma makes it a tuple, not the parentheses.

## 🔶 Immutability — The Core Feature 

Once a tuple is created, its contents cannot be changed. This is not a bug — it is the entire point.

```python
server_config = ("localhost", 8080, "postgresql")

# Trying to change a value
# server_config[1] = 9000    ← TypeError: 'tuple' object does not support item assignment

# Trying to add an item
# server_config.append("ssl") ← AttributeError: 'tuple' object has no attribute 'append'

# Trying to delete an item
# del server_config[0]        ← TypeError: 'tuple' object doesn't support item deletion
```

### But wait — what if the tuple contains a mutable object?
```python
# The tuple itself is immutable but its mutable contents can change
student = ("Ayesha", [88, 92, 75])   # list inside tuple

# Can't replace the list
# student[1] = [90, 95, 80]    ← TypeError

# But CAN modify the list itself
student[1].append(96)           # modifying the list — allowed
student[1][0] = 90              # changing list item — allowed
print(student)                  # ('Ayesha', [90, 92, 75, 96])
```

> 💡 The tuple's **reference** to the list is immutable. The list itself is still mutable.

## 🔶 Accessing Elements

### Indexing — positive and negative:
```python
flight = ("PK-302", "Karachi", "Lahore", "14:30", "16:00")
#           0           1          2         3        4
#          -5          -4         -3        -2       -1

print(flight[0])    # PK-302   ← first item
print(flight[2])    # Lahore   ← third item
print(flight[-1])   # 16:00    ← last item
print(flight[-2])   # 14:30    ← second from last
```

### Slicing — extract a portion:
```python
print(flight[1:3])    # ('Karachi', 'Lahore')  → index 1 and 2
print(flight[:3])     # ('PK-302', 'Karachi', 'Lahore')  → first 3
print(flight[3:])     # ('14:30', '16:00')  → from index 3 onward
print(flight[::2])    # ('PK-302', 'Lahore', '16:00')  → every 2nd item
print(flight[::-1])   # reversed tuple
```

## 🔶 Tuple Unpacking

Unpacking lets you assign each element of a tuple to a separate variable in one clean line.

```python
# Basic unpacking — number of variables must match
employee = ("Hassan", "Senior Developer", 120000)
name, role, salary = employee

print(name)    # Hassan
print(role)    # Senior Developer
print(salary)  # 120000
```

### Unpacking in a loop:
```python
employees = [
    ("Ali",   "Engineer",  85000),
    ("Sara",  "Designer",  72000),
    ("Bilal", "Manager",   91000),
]

for name, role, salary in employees:   # unpacks each tuple
    print(f"  {name:<8} {role:<12} Rs.{salary:,}")
# Output:
#   Ali      Engineer     Rs.85,000
#   Sara     Designer     Rs.72,000
#   Bilal    Manager      Rs.91,000
```

## 🔶 Tuple Methods :

Tuples are immutable so they only have **two built-in methods**:

### `.count()` — how many times does a value appear?
```python
grades = (88, 72, 95, 88, 61, 88, 79, 95)

print(grades.count(88))    # 3  → appears 3 times
print(grades.count(100))   # 0  → not found, no error
```

### `.index()` — where does a value first appear?
```python
cities = ("Karachi", "Lahore", "Islamabad", "Peshawar", "Lahore")

print(cities.index("Islamabad"))   # 2  → found at index 2
print(cities.index("Lahore"))      # 1  → first occurrence only

# cities.index("Quetta")   ← ValueError if not found
# Safe way — check first
if "Quetta" in cities:
    print(cities.index("Quetta"))
else:
    print("Quetta not in tuple")
```

## 🔶 Tuple Operations : 

### Concatenation — join two tuples:
```python
morning_tasks = ("wake up", "exercise", "breakfast")
evening_tasks = ("dinner", "reading", "sleep")

full_day = morning_tasks + evening_tasks
print(full_day)
# ('wake up', 'exercise', 'breakfast', 'dinner', 'reading', 'sleep')
```

### Repetition — repeat a tuple:
```python
separator = ("-",) * 5
print(separator)    # ('-', '-', '-', '-', '-')

checkpoint = ("check",) * 3
print(checkpoint)   # ('check', 'check', 'check')
```

### Membership — check if item exists:
```python
allowed_extensions = (".jpg", ".png", ".gif", ".webp", ".svg")

uploaded_file = "photo.png"
extension = "." + uploaded_file.split(".")[-1]

if extension in allowed_extensions:
    print(f"  {uploaded_file} ✅ allowed")
else:
    print(f"  {uploaded_file} ❌ not allowed")
# Output: photo.png ✅ allowed
```

### Length, min, max, sum:
```python
test_scores = (78, 92, 65, 88, 95, 71, 84)

print(len(test_scores))    # 7
print(min(test_scores))    # 65
print(max(test_scores))    # 95
print(sum(test_scores))    # 573
print(sum(test_scores) / len(test_scores))   # 81.857... average
```

## 🔶 Looping Over Tuples

### Basic for loop:
```python
subjects = ("Math", "Physics", "Chemistry", "Biology", "English")

for subject in subjects:
    print(f"  → {subject}")
```

### With enumerate() — index and value:
```python
for index, subject in enumerate(subjects, start=1):
    print(f"  {index}. {subject}")
# Output:
#   1. Math
#   2. Physics
#   3. Chemistry
#   4. Biology
#   5. English
```

### With while loop — manual index control:
```python
scores = (88, 72, 95, 61, 79)
i = 0

while i < len(scores):
    status = "Pass ✅" if scores[i] >= 70 else "Fail ❌"
    print(f"  Score {scores[i]:>3}  →  {status}")
    i += 1
# Output:
#   Score  88  →  Pass ✅
#   Score  72  →  Pass ✅
#   Score  95  →  Pass ✅
#   Score  61  →  Fail ❌
#   Score  79  →  Pass ✅
```

### Search with break:
```python
suspects = ("Ali", "Sara", "Bilal", "Hina", "Omar")
target = "Bilal"

for i, name in enumerate(suspects):
    if name == target:
        print(f"  Found '{target}' at position {i}")
        break
else:
    print(f"  '{target}' not found")
# Output: Found 'Bilal' at position 2
```

## 🔶 Nested Tuples

A tuple containing other tuples — very common for representing structured fixed data like coordinates, records, or tables.

```python
# 2D grid coordinates
grid_points = (
    (0, 0),
    (1, 0),
    (0, 1),
    (1, 1),
)

for x, y in grid_points:
    print(f"  Point → x:{x}, y:{y}")

# Student database records
students = (
    (101, "Ayesha", "CS",      3.9),
    (102, "Bilal",  "EE",      3.5),
    (103, "Zara",   "Physics", 3.7),
)

print("\n=== Student Records ===")
for roll, name, dept, gpa in students:
    print(f"  {roll}  {name:<8}  {dept:<8}  GPA: {gpa}")
# Output:
# === Student Records ===
#   101  Ayesha    CS        GPA: 3.9
#   102  Bilal     EE        GPA: 3.5
#   103  Zara      Physics   GPA: 3.7
```

## 🔶 When to Use a Tuple vs a List

| | Tuple | List |
|---|---|---|
| **Mutable?** | ❌ No — fixed after creation | ✅ Yes — can change anytime |
| **Syntax** | `(1, 2, 3)` | `[1, 2, 3]` |
| **Speed** | ✅ Faster | ❌ Slightly slower |
| **Memory** | ✅ Less | ❌ More |
| **Methods** | Only `.count()` and `.index()` | 11+ methods |
| **Use when** | Data should never change | Data needs to change |
| **Examples** | GPS, RGB, DB records, config | Shopping cart, student list |

```python
# Use TUPLE — coordinates never change
location = (24.8607, 67.0011)

# Use LIST — cart items change as user adds/removes
shopping_cart = ["biryani", "naan", "lassi"]
shopping_cart.append("karahi")
```

### 🔻 Example 3 — Monthly Sales Report (Read-Only Records)
```python
monthly_sales = (
    ("January",   125000),
    ("February",  98000 ),
    ("March",     143000),
    ("April",     167000),
    ("May",       112000),
)

total   = 0
highest = monthly_sales[0]
lowest  = monthly_sales[0]

print("=== Monthly Sales ===")
for month, sales in monthly_sales:
    bar = "█" * (sales // 10000)
    print(f"  {month:<10} Rs.{sales:>8,}  {bar}")
    total += sales
    if sales > highest[1]:
        highest = (month, sales)
    if sales < lowest[1]:
        lowest = (month, sales)

print(f"\n  Total   : Rs.{total:,}")
print(f"  Best    : {highest[0]} — Rs.{highest[1]:,}")
print(f"  Weakest : {lowest[0]} — Rs.{lowest[1]:,}")
print(f"  Average : Rs.{total // len(monthly_sales):,}")
```

## 🔶 Common Mistakes

### ❌ Mistake 1 — Forgetting the comma for single-item tuple:
```python
wrong = ("hello")        # this is just a string!
right = ("hello",)       # this is a tuple ✅

print(type(wrong))   # <class 'str'>
print(type(right))   # <class 'tuple'>
```

### ❌ Mistake 2 — Trying to modify a tuple:
```python
config = ("localhost", 5432, "mydb")

# config[1] = 9000     ← TypeError ❌

# If you truly need to change it — convert, modify, convert back
config = list(config)
config[1] = 9000
config = tuple(config)
print(config)   # ('localhost', 9000, 'mydb') ✅
```

### ❌ Mistake 3 — Using `.index()` without checking first:
```python
cities = ("Karachi", "Lahore", "Islamabad")

# cities.index("Quetta")   ← ValueError ❌

# Safe way — always check with 'in' first
if "Quetta" in cities:
    print(cities.index("Quetta"))
else:
    print("Not found")    # ✅ no crash
```

### ❌ Mistake 4 — Confusing tuple with list when you need to modify:
```python
# Wrong choice — using a tuple for data that changes
# cart = ("biryani", "naan")
# cart.append("lassi")    ← AttributeError ❌

# Right choice — use a list for data that changes
cart = ["biryani", "naan"]
cart.append("lassi")      # ✅
print(cart)
```

## 🔶 Quick Reference

```
Creating:
  t = (1, 2, 3)              # parentheses
  t = 1, 2, 3                # packing — no parentheses
  t = (42,)                  # single item — trailing comma required
  t = tuple([1, 2, 3])       # from list

Accessing:
  t[0]                       # first item
  t[-1]                      # last item
  t[1:4]                     # slicing

Unpacking:
  a, b, c = t                # each element to a variable
  a, *rest = t               # first + collect the rest
  x, y = y, x                # swap variables

Methods (only 2):
  t.count(x)                 # how many times x appears
  t.index(x)                 # first position of x

Operations:
  t1 + t2                    # concatenate
  t * 3                      # repeat
  x in t                     # membership check
  len(t), min(t), max(t), sum(t)

Looping:
  for item in t:             # basic
  for i, item in enumerate(t): # with index
  for a, b in t:             # unpacking pairs

Tuple vs List:
  Tuple → fixed data, faster, less memory, immutable
  List  → changing data, more methods, mutable
```
